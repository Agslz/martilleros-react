3. PERFILES (local / dev / prod)
================================================================================

local (application-local.yml):
  - BD: localhost:3307 (Docker) o 3306 (MySQL nativo)
  - ddl-auto: validate | open-in-view: false | Flyway: baseline-on-migrate=true
  - Email: MAILHOG (localhost:1025) | Storage: LOCAL (./storage)
  - show-sql: true | Logging: DEBUG (app, hibernate.SQL)
  - Permite: endpoints DEV, MAILHOG, LOCAL storage

dev (application-dev.yml):
  - BD: variables de entorno | ddl-auto: validate | Flyway: baseline-on-migrate=false
  - Email: MAILHOG | Storage: LOCAL | Logging: INFO (app), DEBUG (SQL)
  - Permite: endpoints DEV, MAILHOG, LOCAL storage

prod (application-prod.yml):
  - BD: MySQL con useSSL=true | Flyway: baseline-on-migrate=true (primer deploy)
  - Email: SMTP (credenciales) | Storage: S3 (credenciales AWS)
  - Rechaza MAILHOG y LOCAL (fail-fast en AppPropertiesConfig)
  - Requiere: JWT_SECRET; AWS_* si S3; MAIL_* si SMTP; MERCADOPAGO_ACCESS_TOKEN opcional

  - Java: camelCase | BD: snake_case

6. ENTIDADES PRINCIPALES
================================================================================

Matriculado → matriculados. Login por matricula (UNIQUE). email, password (BCrypt), primeraVezLogin, estadoFianza, role (ADMIN/MATRICULADO). Rel: Fianza, Pago.
Fianza → fianzas. constanciaUrl, fechaVencimiento, fechaInicio. ManyToOne Matriculado.
Subasta → subastas. precioInicial, martilleroACargo, nombreMartillero, cuitMartillero, domicilio, fechaInicio, fechaFin, edictoUrl, visiblePublico. OneToMany ImagenSubasta.
ImagenSubasta → imagenes_subasta. fileName, fileUrl, orden. ManyToOne Subasta.
DocumentoBiblioteca → documentos_biblioteca. titulo, descripcion, fileName, fileUrl, visibleParaMatriculados.
Contenido → contenidos. key (UNIQUE), titulo, cuerpo.
Pago → pagos. tipoPago, monto, estado, descripcion. ManyToOne Matriculado.

Cuotas (Mercado Pago):
CuotaPeriodo → cuota_periodo. periodo (YYYY-MM UNIQUE), monto, fechaVencimiento.
CuotaEstado → cuota_estado. estado (PENDIENTE/PAGADO/VENCIDO), metodoPago (SUSCRIPCION/CUPON_EFECTIVO), paidAt, externalRef. UNIQUE(matriculado_id, cuota_periodo_id).
PaymentOrder → payment_order. provider, type, status, mp_*, init_point, expires_at. FK Matriculado, CuotaPeriodo (nullable).
WebhookEventLog → webhook_event_log. provider, eventType, eventId, payloadJson, processed, errorMessage. UNIQUE(provider, event_id).

9. SEGURIDAD Y JWT
================================================================================

SecurityConfig: CSRF deshabilitado, Session STATELESS.
Filtros: RateLimitingFilter → JwtAuthenticationFilter → UsernamePasswordAuthenticationFilter.

Rutas:
  - permitAll: /api/auth/**, /api/public/**, /api/dev/** (si LOCAL), /swagger-ui/**, /api-docs/**, /actuator/health, /actuator/info, /api/webhooks/**
  - hasAnyRole(ADMIN, MATRICULADO): /api/private/**
  - hasRole(ADMIN): /api/admin/**
  - anyRequest(): authenticated

JWT: Header Authorization: Bearer <token>. Subject = matrícula. Expiración: 86400000 ms (24 h).
Login: matricula + password (no email). Ver LoginRequest.

Rate limiting: POST /api/auth/login 10/min; POST /api/auth/olvide-contrasena 5/min por IP.

POST /api/private/fianzas     [Bearer] multipart file, fechaInicio, fechaVencimiento

GET  /api/dev/files/{folder}/{filename} folders: fianzas, subastas, subastas/edictos, biblioteca

15. PAGO DE CUOTAS (MERCADO PAGO)
================================================================================

Modalidades: Cupón efectivo (initPoint → PagoFácil/Rapipago); Suscripción (débito automático, initPoint adhesión).
Config: MERCADOPAGO_ACCESS_TOKEN (obligatorio para cupón/suscripción), MERCADOPAGO_WEBHOOK_SECRET opcional.
Provider: PaymentProvider (createOfflineCoupon, createSubscription, handleWebhook). MercadoPagoProvider con Orders API y Preapproval API.
Webhooks: POST /api/webhooks/mercadopago; registro en WebhookEventLog; idempotente por event_id; actualiza PaymentOrder y CuotaEstado.
